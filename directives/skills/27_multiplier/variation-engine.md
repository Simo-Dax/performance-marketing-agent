# Variation Engine ,  Andromeda-Compliant Multiplication

This file governs how Claude generates variations that are genuinely different in Meta's eyes. The core principle: copy variation alone is not enough. Each variation must differ in BOTH strategic direction AND visual world. Same scene with different copy equals one Entity ID equals one auction ticket. Different visual scene equals new Entity ID equals new auction ticket.

The user's winning ad is the DNA source. The variations preserve the conversion mechanic and structural skeleton from the analysis but deliberately diverge in scene, color world, hook, and copy.

---

## Step 1 ,  Extract from the VOC Document

Before building the strategy table, extract these five layers from the VOC document:

**Pain point territories** (grouped thematic clusters, not individual complaints):
- Identify 5 to 7 distinct pain point territories
- Tag each with emotional intensity (Low / Medium / High / Extreme)
- Tag each with awareness level
- Note the verbatim customer phrases that anchor each territory

**Desire clusters** (what customers want to feel/achieve/become):
- Identify 5 to 7 distinct desire clusters
- Tag emotional register (aspiration / relief / pride / belonging / transformation)
- Note verbatim phrases

**JTBD dimensions:**
- Functional job: what they're practically trying to do
- Emotional job: how they want to feel
- Social job: how they want to be perceived

**Awareness level distribution:**
- What percentage of quotes are Unaware / Problem-Aware / Solution-Aware / Product-Aware / Most-Aware?
- Which level dominates? Which is underrepresented?

**Language goldmine:**
- Top 10 verbatim phrases with highest emotional intensity
- Top 5 before/after language pairs
- Identity language ("I'm the kind of person who...")

---

## Step 2 ,  Extract from the Brand DNA Document

**Hard constraints** (must apply to every variation):
- Core brand voice descriptors
- Words or phrases never to use
- Claim limitations
- Color palette (primary, secondary, accent)

**Soft constraints** (can flex within a range):
- Tone spectrum (e.g. "confident but warm, can shift between mentor and friend, never corporate or aggressive")
- Photography style range (e.g. "clean studio to warm lifestyle, never dark or gritty")

**Freedom zones:**
- Which emotional registers are explicitly on-brand?
- Which persona types does the brand serve?
- What visual aesthetics reflect the brand?

---

## Step 3 ,  Build the Visual Scene Library

Before mapping copy to visuals, define 8 to 10 distinct visual scenes that work for this product within the brand DNA. The original winning ad's scene is one of them. The other variations must each use a different scene.

Each scene must:
- Place the product in a genuinely different context or environment
- Create a different emotional feeling when seen
- Differ visually enough that Andromeda's computer vision assigns a different embedding

### Scene categories to draw from:

**Environment-based scenes:**
- Morning ritual (bathroom counter, natural light, skincare/coffee/supplement context)
- Kitchen/dining (preparation, ingredients visible, food styling)
- Workspace/desk (productivity context, mid-task placement)
- Gym bag / active (sports kit, water bottle, post-workout)
- Outdoor / lifestyle (park bench, café table, natural setting)
- Bedroom / sleep ritual (nightstand, soft evening light)
- Travel (hotel, suitcase, airport, on-the-go)

**Composition-based scenes:**
- Product hero (clean background, product as sole subject)
- Hand-held (product held by a person in natural light)
- Flatlay (product among complementary objects, overhead)
- In-use (product being actively used in context)
- Ingredient/component explode (product surrounded by key parts)
- Before/after framing (problem context vs result context)
- Social/sharing moment (product being shown to or shared with someone)
- UGC/phone-shot (raw, casual, native-feed feel)

**Color world variations:**
- Light and airy (white, cream, soft morning light)
- Dark and moody (deep backgrounds, dramatic shadows)
- Warm and earthy (wood, terracotta, natural textures)
- Bright and bold (saturated brand colors, energetic)
- Clinical/clean (crisp white, precise lighting)
- Vintage/editorial (grain, muted tones, analog feel)

**Hard rule:** Never use the same scene AND the same color world in two variations. If two variations share a scene category, they must diverge significantly in color world, composition angle, or accompanying props. The original winning ad's scene should also not be re-used in any variation.

## Step 4 ,  Build the Strategy Table

Create a table with one row per variation (5 to 8 rows, matching the count the user requested). Each row specifies:

| # | Copy Angle | Hook Mechanic | Awareness Level | Emotional Register | Visual Scene | Color World | Differentiator (vs original + vs other rows) |
|---|---|---|---|---|---|---|---|

**Rules for the table:**
- No two rows can share the same Hook Mechanic
- No two rows can share the same Awareness Level (unless 8 variations ,  then max 2 rows per level)
- No two rows can share the same Visual Scene
- No two rows can share the same Color World
- No two rows can share the same Emotional Register
- The original winning ad's hook, scene, and color world should not appear in any variation. If the winner is testimonial-led with warm kitchen lighting, no variation should also be testimonial-led with warm kitchen lighting. Variations explore the unused territory.
- At least one row must be Problem-Aware
- At least one row must be Solution-Aware or Most-Aware
- At least one row must use VOC Language Goldmine phrases as the primary copy angle
- At least one row must be identity-led (Unaware stage, no product mention in hook)
- If the user provided an offer with "all variations", every row must include an offer placement. If "mix", roughly half the rows include the offer.

**Hook Mechanic options** (pick one distinct mechanic per row):
- Curiosity gap, creates an information gap that demands resolution
- Bold claim, specific falsifiable result or statistic
- Pattern interrupt, says or shows something unexpected
- Relatability, mirrors the customer's lived experience precisely
- Social proof, adoption numbers or testimonials or peer behavior
- Fear/loss, what they stand to lose by not acting
- Aspiration, paints the desired identity or outcome
- Problem-agitation, names and amplifies the pain
- Direct offer, value-first hook leading with the deal

**Emotional Register options** (pick one distinct register per row):
- Frustration/relief, acknowledges the pain and promises resolution
- Pride/achievement, appeals to desire for accomplishment
- Belonging/community, connects to tribe or peer identity
- Fear/urgency, activates loss aversion
- Aspiration/transformation, paints the future self
- Curiosity/intrigue, makes them want to know more
- Validation/permission, tells them it's okay to want this

## Step 5 ,  The Conversion Mechanic Constraint

This is the rule that distinguishes multiply 2.0 from a generic variation tool.

The Phase 1 analysis identifies the original ad's conversion mechanic (testimonial-led, claim-led, lifestyle-led, etc.). Every variation must HONOR this mechanic even while diverging in scene and angle.

This does NOT mean every variation must use the same mechanic. It means every variation must contain the structural element that the mechanic depends on.

Examples:
- If the original is **testimonial-led**, every variation must contain a testimonial element (a quote, a star rating, an attributed line). The scene changes, the testimonial source changes, but the mechanic stays.
- If the original is **claim-led**, every variation must contain a specific falsifiable claim or stat. The number can change (different angles surface different stats from the VOC), but every ad has one.
- If the original is **lifestyle-led**, every variation must sell the moment or identity more than the product. The moment changes, the identity facet changes, but the lifestyle frame stays.
- If the original is **demo-led**, every variation must show the product in use or its mechanism. The use case changes, the angle changes, but the demo element stays.

This is what preserves the win-rate genetics across variations.

## Step 6 ,  Differentiation Audit

Before finalizing the strategy table, compare every row against every other row AND against the original winning ad. For each comparison, confirm:

**Visual differentiation test:** If you removed all the text, would these two ads look genuinely different to a human viewer? If someone saw both in a feed would they feel like they'd seen different ads? If no, one of them needs a different scene or color world.

**Message differentiation test:** Do these two ads make the same promise to the same person? If yes, one needs a different angle or awareness level.

**Entity ID likelihood:** Would Meta's computer vision (which analyzes color, composition, object placement, and spatial patterns) likely assign different embeddings to these two images? Use this checklist:
- Different dominant color/palette → ✓ new Entity ID likely
- Different background environment → ✓ new Entity ID likely
- Different product positioning (hero vs in-use vs hand-held) → ✓ new Entity ID likely
- Different composition structure (product left vs centered vs right) → ✓ new Entity ID likely
- Only different copy, same visual → ✗ same Entity ID likely

Every variation must pass all three tests against every other variation AND against the original winning ad before the prompts are written.

---

## Step 7 ,  Output Format for the Strategy Table

Present the strategy table to the user like this:

```
Here are the [N] variations I'm planning. Each preserves your winning ad's [conversion mechanic] but uses a genuinely different visual scene and strategic direction so Meta treats them as [N] separate ads.

VARIATION 1
Angle: [which VOC pain point or desire]
Hook: [mechanic]
Awareness level: [stage]
Emotional register: [feeling]
Visual scene: [brief description of scene, lighting, color world]
Why it's different from the original winner: [one sentence]
Why it's different from the other variations: [one sentence]

VARIATION 2
[same structure]

(continue through all variations)

Confirm to proceed and I'll write all [N] complete prompts.
```

Wait for user confirmation before writing the full prompts. If the user wants to adjust any variation (different angle, different scene, swap one out, change the number), revise the table before proceeding.

## Common Mistakes to Avoid

**Mistake 1, synonym swaps disguised as variation**
"Fix your skin" vs "Transform your skin" vs "Heal your skin" are not different variations. They're the same message with different words. The angle must differ. One leads with pain, one leads with identity, one leads with social proof.

**Mistake 2, same visual scene across variations**
Even if the structural skeleton is preserved (which it should be), if every variation has the same kitchen lighting and the same product placement and the same color palette, Andromeda clusters them. Each variation must have a distinct scene AND color world.

**Mistake 3, replicating the original ad's scene in a variation**
The original winning ad already exists. The variations exist to multiply it, not duplicate it. No variation should use the original's scene or color world. Every variation explores fresh visual territory.

**Mistake 4, breaking the conversion mechanic**
If the original wins because of social proof and you write a variation with no social proof element, that variation is a different ad, not a multiplication. It might still work, but it's not testing the same conversion DNA. Stay disciplined.

**Mistake 5, generic ad copy not grounded in VOC**
If the copy doesn't contain verbatim customer language, it sounds AI-written. Andromeda doesn't punish this directly, but real humans do. Every variation's key phrase should be traceable to the VOC document.

**Mistake 6, all variations at the same awareness level**
Running 5 product-aware variations limits reach to people who already know the brand. Spread variations across awareness levels (at minimum: one Problem-Aware, one Solution-Aware, one Product-Aware) to unlock different audience segments.
