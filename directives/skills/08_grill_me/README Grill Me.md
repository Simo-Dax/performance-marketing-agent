# Grill Me — README

## Overview
Requirements handshake attraverso interrogazione strutturata. Previene iterazioni inutili raccogliendo tutti i requirements critici prima di iniziare il lavoro creativo.

## Quick Start
```bash
# Quando usare:
"Devo creare una landing page" [contesto vago]
"Help me build a funnel" [scope non chiaro]

# Output: Requirements Doc completo in 4 fasi (5-10 min di Q&A)
```

## Le 4 Fasi
1. **Context Discovery**: Business model, obiettivo, competitor
2. **Audience Deep Dive**: Chi è, awareness level, obiezioni
3. **Offer & Conversion**: Cosa offri, commitment, funnel
4. **Traffic & Message Match**: Sorgente traffico, messaggio pre-click

## ROI
- 5-10 minuti di domande iniziali = risparmio di ore di rework
- Output allineato al primo tentativo
- Riduzione frustrazione utente e agent

## Integration
Usa come Step 0 quando il brief CRO ha 3+ gap bloccanti, oppure quando l'utente parte con task vago.
